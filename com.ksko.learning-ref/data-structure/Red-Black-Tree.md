레드블랙트리
=
자가균형이진트리. 자료구조의 하나로 키-값으로 이루어져 있어서 map이라고도 함
자료의 삽입,삭제,검색에 일정한 실행시간(시간복잡도 O(log n))을 보장하기 때문에 C++의 map이나 자바의 트리맵도 레드블랙트리를 사용한다.

특성
-
1. 각 노드는 블랙/레드로 구분된다.
2. 루트 노드는 블랙이다.
3. 모든 리프노트(끝노드. null)는 블랙이다.
4. 레드노드의 자식은 블랙이다. 블랙노드의 자식은 레드일수도, 블랙일수도 있다.
	4.2 따라서 레드 노드는 경로의 절반보다 많을 수 없다. 최대 절반이다.
5. 어떤 노드에서 리프노드까지 리프노드를 제외하고 모든 경로에는 블랙노드의 개수는 같다.
6. 트리의 좌우는 균형을 이룬다.

구현
-
#삽입
1. 새로운 노드를 단말노드에 삽입한다. 단말노드와 교체한다.
2. 새로운 노드는 항상 레드이고 그 아래 단말노드(null) 2개가 주어진다.

#삽입 시 문제점
1. double red 
	레드 - 단말노드(블랙) 에서 레드 - 새 노드(레드) 인 경우 발생
	
2. 블랙노드가 다른 경로의 수보다 많다.

#문제점 해결

##레드블랙 재정렬 방식
1. Restructuring 시간복잡도 O(log n)
2. Recoloring 시간복잡도 O(1) / O(log n)
